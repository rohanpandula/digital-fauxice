# Synthetic LaMa example

This directory archives one real IOPaint 1.6.0 / LaMa hybrid run. The input is
deterministically generated by `../make_synthetic_inputs.py`; the infrared
channel contains one 20 x 20 low-IR patch. The output is generative only inside
`run/synth-mask.png` (`255` means synthesized). Outside that mask,
`run/output-hybrid.rgb16.npy` is byte-identical to the pure Portable Digital
ICE output.

The run used CPU inference, one thread, seed request 0, IOPaint source manifest
`64c09870ecefa983d0ba3ceaefc56a5efe7dfcf2a96394548757df3a6ceacb31`,
and external `big-lama.pt` SHA-256
`344c77bbcb158f17dd143070d1e789f38a66c04202311ae3a258ef66667a9ea9`.
The model file is not included. The archived receipt binds hybrid source
manifest
`2d765b0de5ef53e567909f1295d5139065e2a30494e58042cc5207b9c18e4729`
and has SHA-256
`799ce21b54637b4d6da27ad518ef05d69e467c9488f78d1e37764e8db4fbe337`.

An independent same-environment repeat produced identical file hashes for the
pure output, hybrid output, synthesis mask, component input, component mask,
and component inpainted result. The archived run synthesized 840 of 65,536
pixels (1.28173828125%). Key repeat-equal hashes are:

# Security policy

## Reporting a problem

Use GitHub's private vulnerability reporting for security issues. If private
reporting is unavailable, open an issue with a short description and no
sensitive attachments so a private channel can be arranged.

Never attach proprietary scanner software, firmware, personal scans, access
tokens, or private capture data to a public report.

## Supported versions

Security fixes target the latest release on the default branch. This is
research software and has not received a third-party security audit.
